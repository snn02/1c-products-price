# Установка и первый запуск

## Требования

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) (рекомендуется) или pip
- Одна из поддерживаемых IDE: Claude Code, OpenCode, Codex

---

## Установка

### Вариант 1 — через uv (рекомендуется)

```bash
uv pip install git+https://github.com/your-org/1c-products-price.git
```

### Вариант 2 — через pip

```bash
pip install git+https://github.com/your-org/1c-products-price.git
```

### Вариант 3 — локально из исходников

```bash
git clone https://github.com/your-org/1c-products-price.git
cd 1c-products-price
uv pip install -e .
```

---

## Первый запуск

### 1. Создать файл окружения

Скопируйте шаблон и при необходимости отредактируйте:

```bash
cp .env.example .env
```

Параметры по умолчанию работают без изменений.

### 2. Загрузить прайс-лист

```bash
mcp-1c-price-update
```

Команда скачает актуальный прайс с сервера 1С (~580 КБ), распакует и загрузит ~7 300 продуктов в локальную базу данных `data/prices.db`.

Вывод при успехе:

```
Скачивание прайс-листа...
Импорт продуктов: 7329
Дата прайса: 28 апреля 2026
База обновлена за 4.2 сек.
```

### 3. Запустить сервер вручную (опционально, для проверки)

```bash
mcp-1c-price
```

Сервер запускается через stdio-транспорт. В обычном режиме его запускает IDE автоматически.

---

## Обновление прайса

Прайс-лист 1С обновляется ежедневно. Для обновления базы:

```bash
# Через CLI
mcp-1c-price-update

# Или из IDE — попросите AI:
# "обнови прайс"
```

Если прайс не изменился с последнего обновления, повторный импорт пропускается.

## Структура данных

База `data/prices.db` содержит продукты со следующими полями:

| Поле | Описание |
|------|----------|
| `code` | Артикул (13-значный код) |
| `name` | Полное наименование |
| `currency` | Валюта (руб., USD, BYN, KZT) |
| `price_retail` | Рекомендованная цена |
| `price_dealer` | Цена для дилера |
| `price_partner` | Цена для постоянного партнёра |
| `vat` | Ставка НДС |
| `section` | Раздел прайса |
| `date_start` | Дата начала продаж |
| `registry` | Номер в реестре российского ПО |
| `comment` | Комментарий |
