# План разработки v1

## Контекст

MCP-сервер для менеджеров по продажам: пишут свободным текстом какие продукты 1С нужны заказчику → AI находит продукты в локальной базе, формирует КП с кодами, ценами и итогом. Работает в Claude Code, OpenCode, Codex через stdio-транспорт.

Источник данных: `https://1c.ru/ftp/pub/pricelst/price_1c.zip` — XLS с ~9 155 строками, ~7 300 рублёвых продуктов, обновляется ежедневно.

## Шаг 3 — db.py

SQLite + FTS5. Функции:

| Функция | Описание |
|---------|----------|
| `init_db()` | создать таблицы если не существуют |
| `replace_all(products)` | очистить + загрузить все продукты, rebuild FTS индекс |
| `search(query, limit=10)` | FTS5 поиск, fallback на LIKE |
| `get_by_code(code)` | точный поиск по артикулу |
| `meta()` | кол-во продуктов, дата обновления |

**Схема таблицы `products`:**
```
code, name, currency, price_retail, price_dealer, price_partner,
vat, section, date_start, registry, comment, updated_at
```

**Виртуальная таблица `products_fts`:** FTS5 по полю `name`, tokenize=unicode61.

## Шаг 4 — updater.py

Функции:

| Функция | Описание |
|---------|----------|
| `download()` | скачать ZIP через httpx → bytes XLS |
| `parse(xls_bytes)` | распарсить XLS → list[dict] |
| `run()` | download → parse → db.replace_all → {count, price_date, elapsed} |
| `main()` | CLI entry point |

**Логика парсинга XLS (индексы колонок):**
- Col 0: код, Col 1: название, Col 3: валюта
- Col 4: цена розница, Col 5: дилер, Col 6: партнёр
- Col 7: НДС, Col 8: дата продаж, Col 9: реестр, Col 10: комментарий
- Строка — продукт если: col[0] непустой, col[1] непустой, col[4] числовое
- Строка — заголовок раздела если: col[0] содержит "Раздел", col[1] пустой

## Шаг 5 — server.py + __init__.py + __main__.py

FastMCP с четырьмя инструментами. Descriptions на английском.

| Инструмент | Описание |
|-----------|----------|
| `search_products(query, limit=10)` | FTS поиск по названию |
| `get_product(code)` | получить по артикулу |
| `build_quote(items: [{code, qty}])` | сформировать КП, вернуть markdown-таблицу + итог |
| `refresh_prices()` | скачать и обновить базу |

**instructions** (системный промпт для AI):
> When a user mentions products they need, call search_products for each product mentioned, then call build_quote with found codes and quantities. If the database is empty or missing, call refresh_prices first.

## Верификация

### Проверка после каждого шага

| Шаг | Команда проверки |
|-----|-----------------|
| 1 — pyproject.toml | `uv pip install -e ".[dev]"` без ошибок |
| 2 — settings.py | `python -c "from mcp_1c_price import settings; print(settings.DB_PATH)"` |
| 3 — db.py | `pytest tests/test_db.py` |
| 4 — updater.py | `pytest tests/test_updater.py` + `mcp-1c-price-update` (реальная загрузка) |
| 5 — server.py | `fastmcp dev src/mcp_1c_price/server.py` (интерактивный MCP Inspector) |
| 6 — configs | `python -m json.tool configs/claude-code.json` (валидация синтаксиса) |
| 7 — tests | `pytest --tb=short` (все зелёные) |
| 8 — README | установка с нуля по инструкции из README |

### Защита от регрессий

- Тесты пишем **сразу после модуля** — `db.py` → `test_db.py` в том же шаге
- `test_db.py` использует `pytest`-фикстуру с **in-memory SQLite** (`:memory:`) — не трогает реальную базу
- `test_updater.py` парсит **эталонные строки** через fixture — не скачивает реальный XLS
- Перед каждым коммитом `pytest` должен быть зелёным

### Итоговая интеграционная проверка (после Шага 5)

```bash
# 1. Загрузить базу
mcp-1c-price-update

# 2. Поднять сервер в dev-режиме (открывает MCP Inspector в браузере)
fastmcp dev src/mcp_1c_price/server.py

# 3. В MCP Inspector вызвать вручную:
#    search_products("бухгалтерия") → список продуктов
#    build_quote([{"code": "4601546057532", "qty": 1}]) → таблица с итогом
#    refresh_prices() → статус обновления

# 4. Финальный сценарий в IDE:
#    "ERP корп + сервер мини + 5 лицензий, сделай КП"
#    → AI должен вызвать search_products 3 раза, затем build_quote
```
